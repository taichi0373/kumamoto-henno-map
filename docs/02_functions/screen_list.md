## 画面別詳細仕様

### S001: ホーム画面（Home.vue）

**概要**: アプリケーションのメイン画面。地図表示と特典検索機能を統合した中核画面。

**レイアウト構成**:
```
┌─────────────────────────────────────┐
│ ヘッダー（ナビゲーション・ユーザーメニュー） │
├─────────────────────────────────────┤
│ 検索バー（キーワード・フィルター）        │
├─────────────────────────────────────┤
│                                     │
│            地図エリア                  │
│         （特典マーカー表示）             │
│                                     │
├─────────────────────────────────────┤
│ 検索結果リスト（サイドパネルまたは下部）   │
└─────────────────────────────────────┘
```

**主要コンポーネント**:
- ヘッダーナビゲーション
- 検索フォーム
- 地図表示エリア
- 特典マーカー
- 検索結果リスト
- フィルターパネル

**画面要素**:
- **検索バー**: キーワード入力、カテゴリ選択、地域選択
- **地図**: インタラクティブマップ、ズーム・パン操作
- **マーカー**: 特典施設のピン表示、クラスタリング
- **結果リスト**: マッチした特典の一覧表示
- **現在地ボタン**: GPS位置取得・表示
- **フィルターボタン**: 詳細絞り込みオプション

**レスポンシブ対応**:
- **デスクトップ**: 地図とリストの左右分割表示
- **タブレット**: 地図とリストの上下分割表示
- **スマートフォン**: 地図メイン、リストは下部スライドアップ

### S011: ログイン画面（Login.vue）

**概要**: ユーザー認証を行うログイン画面。

**レイアウト構成**:
```
┌─────────────────────────────────────┐
│            アプリロゴ                  │
├─────────────────────────────────────┤
│                                     │
│        ログインフォーム                 │

│     │ パスワード                │     │
```

## 画面間連携

### 認証フロー
1. 未認証ユーザーはゲストとしてホーム画面にアクセス可能
2. プロフィール設定・お気に入り機能は要ログイン
3. ログイン画面から認証後、元の画面に戻る
4. セッション切れ時は自動的にログイン画面に遷移

### 検索フロー
1. ホーム画面で検索条件を入力
2. 地図上にマーカー表示、リストに結果表示
3. マーカーまたはリストアイテムクリックで詳細画面表示
4. 詳細画面から経路案内画面に遷移可能

### 管理者フロー
1. 専用ログイン画面から管理者認証
2. ダッシュボード画面で各管理機能にアクセス
3. 各管理画面で CRUD 操作を実行
4. 操作ログは自動記録

## UI/UX設計原則

### アクセシビリティ
- **WCAG 2.1 AA準拠**: 視覚・聴覚・運動機能に制約があるユーザーへの配慮
- **キーボードナビゲーション**: マウス操作が困難なユーザーへの対応
- **高コントラスト**: 視覚に制約があるユーザーへの配慮
- **読み上げ対応**: スクリーンリーダーでの利用を考慮

### ユーザビリティ
- **直感的なナビゲーション**: 最小クリック数でのゴール到達
- **一貫したUI**: 全画面で統一されたデザインパターン
- **レスポンシブデザイン**: デバイス別に最適化されたレイアウト
- **高速なレスポンス**: 3秒以内の画面遷移

### エラーハンドリング
- **わかりやすいエラーメッセージ**: 原因と対処方法を明示
- **入力支援**: リアルタイムバリデーションとヒント表示
- **フォールバック**: 機能停止時の代替手段提供
