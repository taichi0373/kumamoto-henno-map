## 画面別詳細仕様

### S001: ホーム画面（HomePage.vue）

**概要**: アプリケーションのメイン画面。地図表示と特典検索・経路案内機能を統合した中核画面。

**レイアウト構成**:
```
┌─────────────────────────────────────────────────────────┐
│           ヘッダー（ナビゲーション・ユーザーメニュー）         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│                     地図エリア                           │
│                  （特典マーカー表示）                     │
│                                                         │
├─────────────────────────────────────────────────────────┤
│         サイドバー（検索/経路案内/ユーザー特典タブ）          │
├─────────────────────────────────────────────────────────┤
│      サイドバー下部ボタン（問い合わせ・ライセンス情報）        │
└─────────────────────────────────────────────────────────┘
```

**主要コンポーネント**:
- ヘッダーナビゲーション（AppHeader）
- 地図表示エリア（MapLibre GL JS）
- 特典検索タブ（AppSearchBenefit）
- 経路案内タブ（AppRouteGuidance）
- ユーザー特典タブ（AppUsersBenefit）
- フィードバックモーダル（AppFeedbackModal）: サイドバー下部「問い合わせ」ボタン（`pi pi-envelope`）から起動
- ライセンス情報モーダル（AppLicenseModal）: サイドバー下部「ライセンス情報」ボタン（`pi pi-info-circle`）から起動

**画面要素**:
- **地図**: インタラクティブマップ、ズーム・パン操作
- **マーカー**: 特典施設のピン表示
- **サイドバー**: 特典検索・経路案内・ユーザー特典の3タブ切り替え
- **現在地ボタン**: GPS位置取得・表示
- **店舗マーカー表示切替**: 地図右上に固定配置されるコントロールパネル

**マップ上コントロール（右上固定エリア）**:

| 要素 | 説明 |
|-----|------|
| アイコン | 商店アイコン（`pi pi-shop`） |
| ラベル | 「店舗」 |
| トグルスイッチ | ON/OFF で店舗特典マーカーを表示/非表示 |

- ON: 座標を持つ店舗特典（カテゴリ: BS001）を全件取得しマーカー表示
- OFF: 全ての店舗マーカーを削除
- マーカークリック: ポップアップ表示（特典名・内容・電話番号・住所・詳細URL）
- 検索連動: 特典検索結果に座標付き特典がある場合、検索結果のマーカーを地図に表示し、特典カードクリックで地図をパン＋ポップアップ表示

**レスポンシブ対応**:
- **デスクトップ**: 地図とサイドバーの左右分割表示
- **スマートフォン**: 地図メイン、サイドバーはスライドアップ

### S012: 会員登録画面（SignupPage.vue）

**概要**: 新規ユーザーアカウントを作成する画面。

**画面要素**:
- **ユーザー名入力**: ユーザー名フィールド（最大30文字・一意）
- **パスワード入力**: パスワードフィールド
- **メールアドレス入力**: メールアドレスフィールド（一意）
- **生年月日入力**: 日付選択
- **居住自治体**: 熊本県内市町村の選択
- **免許状況**: 免許所持状況の選択
- **会員登録ボタン**: 登録実行

**バリデーション**:
- 必須項目チェック（ユーザー名・パスワード・メールアドレス）
- ユーザー名・メールアドレス重複チェック（サーバー側）
- パスワード文字数チェック
- メールアドレス形式チェック
- 生年月日の未来日チェック

### S016: パスワードリセット画面（ResetPasswordPage.vue）

**概要**: リセットメール内のURLをクリックしてアクセスする画面。URLに含まれるトークン（クエリパラメータ `token`）で本人確認を行い、新パスワードを設定する。

**URL**: `/reset-password?token=<リセットトークン>`

**画面要素**:
- **新しいパスワード入力**: 新パスワードフィールド
- **新しいパスワード（確認）入力**: 確認用フィールド
- **変更ボタン**: パスワードリセット実行

**バリデーション**:
- 必須項目チェック
- パスワードの文字数チェック
- パスワードと確認用パスワードの一致チェック
- トークン有効期限チェック（30分・サーバー側）

## 画面間連携

### 認証フロー
1. 未認証ユーザーはゲストとしてホーム画面にアクセス可能
2. プロフィール設定・パスワード変更機能はログイン必須
3. ログイン画面から認証後、元の画面に戻る（`redirect` クエリパラメータ）
4. セッション切れ時は自動的にログイン画面に遷移

### パスワードリセットフロー
1. ログイン画面から「パスワードをお忘れの方」リンクをクリック
2. メールアドレスを入力してリセットメールを送信
3. メール内のURLをクリックしてリセット画面に遷移
4. 新パスワードを設定してログイン画面に遷移

### 検索フロー
1. ホーム画面で検索条件を入力
2. 地図上にマーカー表示、サイドバーに結果表示
3. マーカーまたはリストアイテムクリックで情報確認
